# 05 字体系统

## 原则

- App UI 使用 SF Pro。
- 内容排版与界面排版是两个子系统。
- 普通 UI 必须支持 Dynamic Type。
- 计时器、时长和速度使用等宽数字。
- 页面不得随意硬编码字号。

## UI Typography Tokens

| Token | 建议 | 用途 |
|---|---:|---|
| Display | 34 | 极少使用的重要首屏标题 |
| Title 1 | 28 | 页面大标题 |
| Title 2 | 22 | 主要分区标题 |
| Title 3 | 20 | 卡片或 Sheet 标题 |
| Headline | 17 Semibold | 行标题、重要操作 |
| Body | 17 Regular | 主要正文 |
| Callout | 16 | 补充说明 |
| Subheadline | 15 | 次级信息 |
| Footnote | 13 | 元数据、帮助说明 |
| Caption | 11–12 | 紧凑标签 |
| Numeric | 15–20 Monospaced | 时间、速度、计数 |

## 字重

默认只使用 Regular、Medium、Semibold，Bold 极少使用。粗体用于层级，不用于制造高级感。

## 提词排版

提词文字是独立系统，需要定义字号、行高、段间距、阅读宽度、眼线区域、已读/当前/未读层级、语音跟随高亮、横竖屏和镜像规则。

建议默认：

- 字号 46–52 pt，按设备调整；
- 行高为字号的 1.35–1.5 倍；
- 段间距为 0.45–0.7 行；
- 竖屏左右边距 24–36 pt；
- 中文单行约 12–20 字，英文约 35–55 字符。

## 禁止事项

- 单页出现 6 种以上字号；
- 为适配空间任意缩小文字；
- 普通文本使用低对比浅灰；
- 大量使用 Rounded Font 造成娱乐化气质。
