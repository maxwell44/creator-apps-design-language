# 09 动效系统

## 原则

动效解释变化，不装饰页面。它应帮助用户理解什么出现、什么消失、什么状态改变，以及操作是否成功。

## Motion Tokens

| Token | Duration | 用途 |
|---|---:|---|
| instant | 0–0.10s | 按压态、即时切换 |
| quick | 0.15s | 图标和轻量反馈 |
| standard | 0.25s | 大多数控件状态 |
| emphasized | 0.35s | Toolbar 展开、模式切换 |
| slow | 0.45s | 少量沉浸式过渡 |

超过 0.45s 的交互动画需要明确理由。

## 核心动效

- Tap Feedback：轻微缩放或亮度变化，0.10–0.15s。
- Toolbar Show/Hide：透明度加轻微位移，不推动脚本文本。
- Record Start：倒计时、录制按钮和状态清晰转换，位置保持稳定。
- Success：短暂、可信，并明确结果保存位置。

## Reduce Motion

开启后，位移动画改为淡入淡出，取消视差、连续浮动和大幅缩放，保留必要状态反馈。

## 禁止事项

- 页面加载时每张卡片延迟入场；
- 无关元素同时 Morph；
- 录制中多个控件持续脉冲；
- 动画导致文本位置改变；
- 为了顺滑而延迟操作结果。
