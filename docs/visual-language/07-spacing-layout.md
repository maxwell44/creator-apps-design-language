# 07 间距与布局

## 基础尺度

采用 4 pt 基准：4、8、12、16、20、24、32、40、48。

## 屏幕边距

- 紧凑 iPhone：16 pt。
- 常规 iPhone：20 pt。
- 阅读型内容：24 pt 起。
- iPad：使用最大内容宽度，不无限拉伸。

## 垂直节奏

- 同组行间：8–12 pt。
- 同组组件间：12–16 pt。
- 分组之间：24–32 pt。
- 页面标题与首个内容：16–24 pt。

## 触控尺寸

所有交互目标至少 44×44 pt。视觉图标可以更小，但 Hit Area 不得缩小。

## Layout Rules

内容区域先确定，控制层在其外围或上方布置。提词与录制页面中，控制出现或消失不得改变脚本文本眼线位置。

横屏不是竖屏的简单旋转，需要重新定义控制位置、安全区、文本宽度、镜像状态和相机遮挡。

Dynamic Type 下允许行布局重排、次级信息换行和 Sheet 高度增加，不允许通过压缩字号维持原布局。

## 禁止事项

- 用固定屏幕高度决定所有面板；
- 用 `UIScreen.main.bounds` 作为设计规范；
- 不同页面使用不同屏幕边距；
- 控件出现导致核心内容跳动。
