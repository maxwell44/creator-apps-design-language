# 06 色彩系统

## 色彩职责

色彩分为四类，禁止混用：

1. 品牌色：主操作与品牌识别。
2. 脚本色：分类、视觉记忆与组织。
3. 状态色：录制、警告、错误与成功。
4. 表面色：背景和层级。

## 品牌色

品牌色保持中等饱和度，只用于主按钮、选中态与关键进度。现有绿色可作为品牌基因，但需要重新校准浅色、深色和对比度，不再铺满页面或承担成功状态。

## Script Palette

彩色脚本是灵感提词器的核心能力，不得取消。

每种颜色至少定义：Light Surface、Dark Surface、Accent、Stroke、Primary Text、Secondary Text、Selected 与 Accessibility Marker。

首页优先使用低饱和有色表面和清晰色标，不使用高饱和整卡渐变。颜色之外增加名称、图标或标签，支持 Differentiate Without Color。

建议保留 8–10 个可区分色系：Coral、Amber、Lemon、Mint、Green、Aqua、Blue、Lavender、Rose、Neutral。

## 状态色

- Recording：红色，只用于正在录制或停止录制。
- Warning：橙/黄，用于可恢复风险。
- Destructive：系统红，用于不可逆动作。
- Success：绿色，但不得与品牌色混淆。
- Info：蓝色，用于系统说明。

## 禁止事项

- 设置页每行使用不同装饰色图标；
- 品牌绿、脚本绿和成功绿没有区分；
- 高饱和渐变作为默认大面积背景；
- 仅靠颜色表达脚本类别；
- 使用颜色掩盖信息层级不足。
