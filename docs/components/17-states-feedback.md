# 17 状态与反馈

## Loading

加载反馈匹配等待时长：300ms 内通常不显示；300ms–1s 使用轻量 Activity；更长任务显示进度、阶段和可取消性。

## Empty State

说明当前状态、用户可以做什么，并提供一个明确主操作。不要用大型插画占据主要空间。

## Error

错误信息必须说明什么失败、用户内容是否安全、下一步能做什么。录制和导出错误优先保护原始文件。

## Success

成功反馈回答是否完成、保存到哪里和下一步是什么。动画短暂，不阻挡用户。

## Permission

说明为什么需要权限、不授权会影响什么、如何稍后开启。不要威胁或强迫用户。

## Recording State

必须清晰区分 Ready、Countdown、Recording、Stopping、Processing、Saved 和 Error。录制状态不能只依赖红色。
