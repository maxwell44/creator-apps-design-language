# 19 首页规范

## 目标

首页是脚本工作区，不是营销页，也不是简单文件列表。用户应快速找到、创建、导入、打开和管理脚本。

## 信息层级

1. 页面标题与搜索。
2. 主要创建入口。
3. 筛选/排序。
4. 脚本内容对象。
5. 次级状态与说明。

## 彩色脚本

彩色脚本是首页核心识别能力。必须保留多个颜色，降低饱和度、提高可读性，在 Light/Dark 下分别定义，并提供颜色之外的辅助标识。

不使用相同白卡加小圆点弱化颜色，也不使用过度渐变。

## 推荐结构

- Large Title 与系统 Search；
- 一个明确 Primary 创建动作；
- 模板与导入为次级入口；
- Script Object 展示标题、2–4 行预览、最近编辑或预计时长、颜色与更多菜单。

搜索、收藏和颜色筛选可作为增量能力，不应成为本轮 UI 重构的阻塞条件。

## 空状态

> 创建第一篇脚本，开始一次更轻松的录制。

主按钮：创建脚本。次级入口：从文件导入 / 使用模板。

## 不做

- 不把首页改成 Dashboard；
- 不增加无关数据统计卡；
- 不用大型渐变营销卡；
- 不取消彩色脚本。
